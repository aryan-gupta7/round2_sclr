## Required files inside `envs/recall_env/` (mandated by OpenEnv)

The OpenEnv CLI (`openenv init`) generates exactly this skeleton. **Do not add or rename top-level files in this folder.** Adding files in `server/` is fine.

| File | Purpose | Notes |
|------|---------|-------|
| `__init__.py` | Re-export public symbols | Must export `RecallAction`, `RecallObservation`, `RecallState`, `RecallEnv` |
| `client.py` | The `RecallEnv` HTTP/WebSocket client | Subclass of OpenEnv's `EnvClient` / `HTTPEnvClient` |
| `models.py` | Pydantic models for Action / Observation / State | These are the wire types — see `03_ENVIRONMENT_SPEC.md` |
| `openenv.yaml` | Manifest read by HF Spaces / CLI | Use the **current schema** (typed, with `runtime`, `image`, etc.). Do **not** copy from older envs that have legacy `name/version/action/observation` shape. |
| `pyproject.toml` | Python deps via uv | Include `openenv-core`, `sentence-transformers`, `pydantic`, `fastapi` |
| `uv.lock` | Generated by `uv lock` | Commit this. |
| `server/app.py` | FastAPI app | Must use `create_app(<factory>, RecallAction, RecallObservation, env_name="recall")`. Pass a class or factory callable, **not** an instantiated environment. |
| `server/recall_environment.py` | Core `Environment` subclass | Implements `reset()`, `step()`, `state` property |
| `server/Dockerfile` | Multi-stage build | Must build `FROM openenv-base` (or use the template-provided base). |
| `server/requirements.txt` | Pinned deps for Docker layer | Mirror what's in pyproject.toml plus runtime |

## Critical OpenEnv conventions

These are pulled from the official `generate-openenv-env` skill. Violating them breaks deployment.

1. **Use typed models.** `RecallAction`, `RecallObservation`, `RecallState` must be Pydantic models extending OpenEnv base classes.
2. **Use `create_app(...)` in `server/app.py`.** Pass a factory or class — do not instantiate the env at import time.
3. **Dual-import pattern.** In `server/app.py` and `server/recall_environment.py`:
   ```python
   try:
       from ..models import RecallAction, RecallObservation, RecallState
   except ImportError:
       from models import RecallAction, RecallObservation, RecallState
   ```
   Relative imports work when running in-repo (`PYTHONPATH=src:envs`); bare imports work in Docker (`PYTHONPATH=/app/env`). Both must work.
4. **Do NOT set `max_concurrent_envs > 1`** unless we explicitly support concurrent sessions. We don't. Leave at 1.
5. **Reserved tool names**: `reset`, `step`, `state`, `close`. Do not use these as MCP tool names.
6. **`openenv.yaml` schema**: Use the current typed schema. Reference `envs/echo_env/openenv.yaml` or `envs/snake_env/openenv.yaml` from upstream. **Do not** copy from `envs/dipg_safety_env` or other older envs with legacy manifests.
7. **Dockerfile**: Build `FROM` the openenv base image so shared tooling (uvicorn, FastAPI wrappers) is available. The default Dockerfile produced by `openenv init` is fine; only modify the deps install section.

## Bootstrap procedure (the very first commit)

A teammate runs:

```bash
pip install openenv-core
openenv init recall_env --output-dir envs/
```

This generates the skeleton. **Do not modify the skeleton mechanics** — only fill in the bodies of `recall_environment.py`, `models.py`, `client.py`, etc. Keep `openenv.yaml` close to the generated one; only edit metadata fields and runtime config.

Then validate:

```bash
cd envs/recall_env
openenv validate
```

This must pass before any further work.

## Per-folder CHANGELOG.md rule

Every folder containing code (`envs/recall_env/`, `baselines/`, `training/`, `tests/`) must contain a `CHANGELOG.md`. See `02_AGENT_RULES.md` for format.
