# Deployment

> **Scope**: Pushing the env to HF Spaces. Required for hackathon submission.
> **Implementation lives in**: `envs/recall_env/openenv.yaml`, `envs/recall_env/server/Dockerfile`, plus the `openenv push` workflow.

## Submission requirements (from hackathon brief)

1. **OpenEnv-compliant environment** hosted on HF Spaces
2. **README** with motivation, env explanation, results — links to env URL, Colab, blog/video, plots
3. **Plots committed as PNGs** to the repo
4. **Training script** as Colab notebook
5. **<2 min video OR mini-blog OR slide deck** linked from README

The env URL is what judges pull. Commits after submission deadline don't count.

## Pre-deploy checklist

Before running `openenv push`:

- [ ] `openenv validate` passes locally
- [ ] All tests in `tests/` pass
- [ ] `envs/recall_env/README.md` is filled out (see template below)
- [ ] `envs/recall_env/openenv.yaml` has correct metadata
- [ ] Smoke test on Docker build succeeds locally
- [ ] At least one trained checkpoint is saved and referenced

## `openenv.yaml` template

Use the schema generated by `openenv init` as the base. **Do not copy from old envs**. The current schema (OpenEnv 0.2+) looks like:

```yaml
# envs/recall_env/openenv.yaml
schema_version: 1
name: recall
display_name: "RECALL — Memory-Constrained Long-Horizon Memory"
description: >
  An OpenEnv environment in which the agent must learn what to store from a stream of
  arriving facts under a tight budget, and answer queries that depend on retained memory.
  The agent's actions include storing items with self-authored retrieval anchors, skipping
  facts, deleting items, retrieving from memory, and answering. Used to train memory
  management policies via RL.

version: "0.1.0"
license: "Apache-2.0"
authors:
  - name: "Suryansh"
    affiliation: "IIT Roorkee"

runtime:
  type: docker
  dockerfile: server/Dockerfile
  port: 8000

action_type: RecallAction
observation_type: RecallObservation
state_type: RecallState

max_concurrent_envs: 1   # do NOT increase — we don't support concurrency

reset_kwargs_schema:
  difficulty:
    type: integer
    minimum: 1
    maximum: 5
    default: 1
  seed:
    type: integer
    minimum: 0
    default: 0

tags:
  - long-horizon
  - memory
  - rl
  - llm
  - benchmark
```

**Verify against an upstream env's `openenv.yaml`** before push. If `openenv validate` complains, match the structure of `envs/echo_env/openenv.yaml` from upstream.

## Dockerfile (server)

The default generated by `openenv init` is correct. Only modify if adding deps. Key points:

```dockerfile
# Multi-stage build using openenv-base
ARG OPENENV_BASE_IMAGE=ghcr.io/meta-pytorch/openenv-base:latest
FROM ${OPENENV_BASE_IMAGE} AS base

WORKDIR /app/env

COPY pyproject.toml uv.lock ./
RUN uv sync --frozen

COPY server/ ./server/
COPY models.py client.py __init__.py openenv.yaml ./

ENV PYTHONPATH=/app/env

CMD ["uvicorn", "server.app:app", "--host", "0.0.0.0", "--port", "8000"]
```

If sentence-transformers needs to be pre-warmed (download model at build time, not first request), add:

```dockerfile
# Pre-download embedding model so first request is fast
RUN python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('all-MiniLM-L6-v2')"
```

This adds ~80 MB to image size but makes startup snappy.

## Push procedure

```bash
cd envs/recall_env
openenv validate                               # must pass
openenv push --repo-id <your-username>/recall-env --enable-interface
```

`--enable-interface` adds a Gradio web UI at `/web` that judges can poke at.

If push fails:
- Auth: `huggingface-cli login`
- Manifest: re-run `openenv validate`, fix errors
- Docker build: test locally with `docker build -t recall-env -f server/Dockerfile .`

## Post-push smoke test

```python
import asyncio
from envs.recall_env import RecallEnv, RecallAction

async def main():
    async with RecallEnv(base_url="https://<your-space>.hf.space") as env:
        obs = await env.reset(difficulty=1, seed=0)
        print("Reset OK:", obs.phase)
        action = build_random_valid_action(obs)
        result = await env.step(action)
        print("Step OK:", result.observation.phase, "reward:", result.reward)

asyncio.run(main())
```

Run this **immediately after push**. If it fails, the env isn't useful to judges.

## What goes in the top-level (project root) `README.md`

Different from the env README. The project README is what reviewers see on GitHub. It must contain:

1. The pitch (one paragraph)
2. Link to the deployed HF Space (the env URL)
3. Link to the Colab training notebook
4. Embedded plots (or links to them)
5. Link to mini-blog / video / slides
6. Quick local-run instructions
7. Citation block (cite RLM, MemGPT, Generative Agents, GraphRAG)

A reviewer should be able to read this README in 3–5 minutes and want to try the env.

## Don't put large files in the env repo

The hackathon brief explicitly says: don't include big video files in the HF Spaces repo. Use external URLs (YouTube for video, Medium / HF Posts for blog).
